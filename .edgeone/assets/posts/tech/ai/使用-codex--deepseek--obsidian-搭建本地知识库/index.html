第一章：环境准备 🛠️

1.1 注册 DeepSeek API

DeepSeek 提供了与 OpenAI/Anthropic 兼容的 API 格式，可通过 Codex++ 插件接入。

flowchart LR
    A[访问 platform.deepseek.com] --> B[注册账号]
    B --> C[创建 API Key]
    C --> D[充值（按量付费）]
    D --> E[获取 API Key 和 Base URL]
    
    style E fill:#4FC3F7,color:#000

关键信息速查：

项目值
官网https://platform.deepseek.com
Base URLhttps://api.deepseek.com
推荐模型deepseek-v4-flash（快速）、deepseek-v4-pro（深度）
API 格式兼容 OpenAI / Anthropic 格式

1.2 安装 Obsidian

如果你还没安装 Obsidian

  1. 从 https://obsidian.md 下载安装
  2. 创建或打开一个 Vault（知识库目录）
  3. 建议安装以下核心插件：
    • 图谱视图（Graph View）— 可视化笔记关联
    • 标签面板（Tag Pane）— 标签管理
    • 反向链接面板（Backlinks）— 查看引用关系
  4. 本教程 Vault 路径示例：C:\Users\用户名\Documents\obisdian

第二章：通过 Codex++ 接入 DeepSeek 🚀

2.1 DeepSeek 模型选择

DeepSeek 提供多种模型，根据使用场景选择：

模型名特性适用场景
deepseek-v4-flash快速响应、低成本日常问答、文档整理、代码生成
deepseek-v4-pro深度推理、更高质量复杂分析、长文写作、架构设计
deepseek-chat兼容旧版（2026/07 弃用）—
deepseek-reasoner旧版推理模型（2026/07 弃用）—

推荐组合

日常使用 deepseek-v4-flash 保持高效响应和低成本，遇到复杂任务（如架构设计、深度分析）时切换到 deepseek-v4-pro。

2.2 安装 Codex++ 插件

Codex++ 是什么？

Codex++ 是一个 Codex 插件，它的核心作用是：

  1. 模型接入 — 将 DeepSeek 等第三方模型桥接到 Codex 平台
  2. 技能管理 — 加载自定义技能（如 Claudian 角色）
  3. 工具扩展 — 提供插件系统、个人市场等高级功能

简单说，Codex 是”躯壳”，Codex++ 是”连接器”，DeepSeek 是”大脑”。

安装步骤：

    B --> C{安装方式}
    C -->|内置市场| D[一键安装]
    C -->|个人插件| E[从 .codex-plugin/ 加载]
    D --> F[配置 DeepSeek<br/>API Key 与模型]
    E --> F
    F --> G[✅ 安装完成]

2.3 配置 Codex++ 连接 DeepSeek

Codex++ 安装完成后，会出现两个入口工具：

工具作用

Codex++ 管理工具配置工具，用于选择和管理大模型供应商

配置步骤

Step 1：打开供应商配置

打开 Codex++ 管理工具 → 点击 供应商配置 → 选择 添加供应商

Step 2：添加 DeepSeek 配置

!90-attachments/Pasted image 20260531153208.png

上图是 Codex++ 管理工具中的 供应商配置界面，图中展示了以下关键信息：

  1. 左侧搜索栏（🔍）— 可搜索已安装的模型供应商
  2. 供应商列表 — 当前可选的 AI 模型供应商，包括：
    • Anthropic（当前选中）— 图标为绿色，配置 claude-sonnet-4-5 模型
    • DeepSeek（需选择）— 点击即可切换到 DeepSeek 模型
    • 其他可选：OpenAI、Google、Grok、Open AI Compatible Provider 等
  3. 选中状态 — 蓝色高亮表示当前激活的供应商
  4. 配置区域 — 选中供应商后，右侧会显示该供应商的详细参数配置项
Step 3：填写 DeepSeek API 信息

选择 DeepSeek 后，在右侧配置区域填写：

配置项值说明
API Key$DEEPSEEK_API_KEY从 platform.deepseek.com 获取，建议通过环境变量管理
Base URLhttps://api.deepseek.comDeepSeek API 固定端点
默认模型deepseek-v4-flash日常使用，快速响应、低成本
备用模型deepseek-v4-pro复杂任务时手动切换，深度推理能力更强
Step 4：保存并验证
  1. 保存配置后，使用 Codex++ 工具启动 Codex
  2. 发送一条简单消息测试链路是否打通： 
    用 Markdown 输出 "Hello World"
  3. 如果 DeepSeek 正确返回 Markdown 格式内容，说明配置成功

配置要点

  • 🔑 API Key 安全 — 建议存放在环境变量 $DEEPSEEK_API_KEY 中，避免硬编码
  • 🔄 闪退降级 — 日常使用 deepseek-v4-flash，复杂任务切换到 deepseek-v4-pro
  • 🔌 协议透明 — Codex++ 在底层使用 OpenAI 兼容协议连接 DeepSeek，用户无需关心协议细节
  • ⚙️ 切换供应商 — 如需切换回 Claude 等其他模型，在同一个界面中重新选择即可

注意：如果打开后仍然未能看到 DeepSeek 模型的选项，请关闭代理后重启应用，等成功进入后再开启代理即可。一定要使用 Codex++ 打开。

第三章：Claudian 安装与配置 ⚙️

3.1 Claudian 安装

Claudian 是 Obsidian 社区的一款插件，能将 AI Agent 能力直接集成到 Obsidian 中，让你在笔记软件内就能与 AI 交互。

安装步骤：

  1. 打开 Obsidian，点击左下角 设置（Settings） !90-attachments/Pasted image 20260531154629.png

    上图是 Obsidian 设置入口界面。点击左下角的齿轮图标 ⚙️ 即可进入设置页面，这是安装第三方插件的第一步。

  2. 选择 第三方插件（Community plugins） !90-attachments/Pasted image 20260531154659.png

    上图是 Obsidian 设置页面的左侧菜单。在”第三方插件”（Community plugins）分类下，可以管理 Obsidian 社区提供的所有扩展插件。如果菜单中没有此选项，请确认你的 Obsidian 版本为 v0.15.0 以上。

  3. 关闭 安全模式（Restricted mode） !90-attachments/Pasted image 20260531154755.png

    上图是第三方插件管理界面。Obsidian 默认启用”安全模式”（Restricted mode），关闭后才可安装社区插件。点击右侧的 Turn off restricted mode 按钮即可。若曾安装过插件，此处还会显示已安装的插件列表。

  4. 点击 浏览社区插件市场（Browse），搜索 Claudian !627

    上图是社区插件市场界面。在搜索栏中输入 “Claudian” 即可定位该插件。请注意选择名为 Claudian 的插件（作者为 realclaudian 团队），避免误装其他同名工具。

3.2 Claudian 配置 Codex

安装完成后，需要配置 Claudian 与 Codex 的连接：

  1. 在已安装插件列表中找到 Claudian，点击 设置 进入配置页面 !90-attachments/Pasted image 20260531155246.png

    上图是 Obsidian 的第三方插件设置页面。在”已安装插件”列表中找到 Claudian，点击右侧的 设置（齿轮图标）进入 Claudian 的专属配置页面。此处可配置 Codex CLI 连接路径、模型供应商等关键参数。

  2. 将 Enable Codex provider 开关打开，并在 Codex CLI path 中填入 Codex CLI 的路径 !90-attachments/Pasted image 20260531155421.png

    上图是 Claudian 插件设置页面的核心配置区域：

    1. Enable Codex provider — 将此开关切换为开启状态（蓝色），这是让 Claudian 通过 Codex 调用 AI 模型的关键开关
    2. Codex CLI path — 在此输入框中填入 Codex 命令行工具的完整路径（如 C:\Users\用户名\AppData\Roaming\npm\codex.cmd）

    如果不知道路径或未安装 CLI，可在 PowerShell 中执行以下命令安装：

    npm install -g @openai/codex

    命令行工具安装完成后会自动同步 Codex 图形化设置，无需额外配置。

    如果提示没有 npm 命令，请先安装 Node.js（访问官网下载安装即可）。

    安装完成后，在命令行执行以下命令查找 Codex 路径：

    where.exe codex

    输出结果中以 .cmd 结尾的那一行即为所需路径。 !90-attachments/Pasted image 20260531160401.png

    上图是 PowerShell 中执行 where.exe codex 命令的输出示例。输出通常包含两条或更多路径，应选择其中以 .cmd 结尾的那一条（如 C:\Users\用户名\AppData\Roaming\npm\codex.cmd），将其完整复制到第 2 步的 Codex CLI path 输入框中。

  3. 配置保存完成后，点击左侧栏的小机器人图标即可开始使用，支持手动切换模型 !90-attachments/Pasted image 20260531160552.png

    上图是 Claudian 聊天界面的侧边栏。左侧的 小机器人图标 🤖 即为 Claudian 的入口。点击后可在右侧打开聊天窗口，顶部的下拉菜单支持在 DeepSeek 等配置好的模型间手动切换。

提示 如果配置未生效，请尝试打开一个新的聊天窗口后再使用。

4.3 Claudian vs 通用 AI 助手的区别

对比维度通用 AI 助手Claudian（Obsidian 专家）
文件操作偶尔可用精通 Markdown、Frontmatter、Wiki-link
知识库理解无上下文感知 Vault 结构、标签、链接网络

面试导向一般回答标注考点、整理常见题、情景排查

4.4 在 Claudian 中配置 Codex 的核心规则

Claudian 之所以能在 Obsidian Vault 中精准操作，是因为在 Codex 中定义了一套系统指令（System Prompt）。这套指令是 Claudian 行为的”宪法”，规定了它如何理解 Codex 的环境、如何操作文件、如何与用户交互。

配置载体

Claudian 的配置通常通过 SKILL.md 文件定义，该文件被 Codex++ 插件加载后生效：

.codex/skills/.system/.../SKILL.md

核心配置模块

以下是在 Codex 中配置 Claudian 所需定义的六大核心规则模块：

模块指令内容作用
⏰ 时间上下文使用 bash: date 获取当前时间，禁止猜测确保文件创建时间、更新时间的准确性
🧑‍💼 身份与角色定位为”Obsidian 知识管理专家”定义 Claudian 的回答风格和行为边界
📁 路径约定使用相对路径，禁止绝对路径避免文件操作错误，确保跨平台兼容
📝 Obsidian 原生语法支持 Frontmatter、Wiki-link、Callout、Mermaid、Dataview让输出内容直接在 Obsidian 中可用
🔗 响应格式回答中使用 [[]] 格式引用文件用户可点击链接直接跳转到对应笔记
🛡️ 安全规则先读后写、不覆盖数据、操作前确认保护用户数据安全

路径约定详解

这是 Claudian 配置中最容易出错的部分，需要精确定义：

✅ 正确写法（相对路径）：
  - notes/my-note.md
  - my-note.md
  - folder/subfolder/file.md

❌ 错误写法（绝对路径）：
  - /notes/my-note.md              ← 前导斜杠
  - C:\Users\...\obisdian\file.md  ← 盘符绝对路径

工作目录：
  Vault根目录 = 当前 Obsidian Vault 路径

响应格式规则

Claudian 在回答中引用文件时遵循以下格式：

✅ 正确：
  - "参见2024-01-15 的日常笔记"     ← 用户可点击跳转
  - "这是 [[AI名词释义]] 中的内容"
  - "![[attachments/architecture.png]]"    ← 图片直接渲染

❌ 错误：
  - "参见 daily notes/2024-01-15"          ← 纯文本，不可点击
  - "参见 /daily notes/2024-01-15"         ← 绝对路径

上下文感知配置

Claudian 配置了三种上下文注入模式，Codex 会自动将当前环境信息传给 Claudian：

<!-- 用户当前打开的笔记 -->
<current_note>
path/to/note.md
</current_note>

<!-- 用户选中的文本 -->
<editor_selection path="path/to/file.md" lines="10-15">
选中的文本内容...
</editor_selection>

<!-- 用户从浏览器选中的内容 -->
<browser_selection source="browser:https://..." title="页面标题" url="...">
选中的网页内容
</browser_selection>

这三种上下文让 Claudian 能够：

  • 知道用户在哪个笔记中，提供上下文相关的建议
  • 精准定位用户选中的文本段落
  • 处理从外部网页抓取的内容

工具权限配置

Claudian 在 Codex 中被授予以下工具调用权限：

工具权限配置说明
shell_command✅ 可用执行文件操作、运行命令（如 Get-ChildItem）
Read✅ 可用读取 Markdown 文件和图片内容
Write✅ 可用创建和编辑笔记
MCP Resources✅ 可用访问 MCP 服务器提供的资源
View Image✅ 可用查看并分析本地图片
删除操作⚠️ 需确认需用户显式确认后才执行

一句话总结配置思路

Claudian 配置的核心公式

Claudian = Codex 工具能力 × DeepSeek 推理能力 × Obsidian 格式规范 × 安全操作规则

配置的目标是让 AI 在正确的路径上用正确的格式做正确的事情，同时不破坏用户数据。

第五章：连接 Codex++ + Obsidian 🔗

5.1 授予 Vault 访问权限

在 Codex 中，需要将 Obsidian Vault 目录设置为工作目录：

Vault 路径：C:\Users\用户名\Documents\obisdian

Codex++ 通过 Codex 的文件系统权限来访问 Vault，支持以下操作：

操作权限示例
读取文件✅读取笔记内容
创建文件✅创建新 Markdown 笔记
编辑文件✅优化文档结构、添加链接
删除文件⚠️ 需确认需用户确认后才执行
创建文件夹✅组织知识库结构

5.2 观察者模式：<current_note>

当用户在 Obsidian 中打开一篇笔记时，Codex 的上下文会自动包含该笔记的信息：

<current_note>
path/to/当前笔记.md
</current_note>

这使得 Claudian 能够：

  • 理解用户正在查看的内容
  • 基于当前笔记提供上下文相关的建议
  • 在回答中引用当前笔记中的内容

5.3 选区模式：<editor_selection>

当用户选中一段文本时，Claudian 能获取到选区的具体内容和位置：

<editor_selection path="path/to/file.md" lines="10-15">
选中的文本内容...
</editor_selection>

这让你可以：

  • 要求对特定段落进行改写/翻译
  • 选中代码块后请求解释
  • 对特定内容添加注释或链接

5.4 浏览器选择：<browser_selection>

当从 Obsidian 内置浏览器（如 Surfing 插件）中选中内容时：

<browser_selection source="browser:https://..." title="页面标题" url="...">
选中的网页内容
</browser_selection>

用于：

  • 抓取网页技术文档转化为笔记
  • 保存论坛问答到知识库
  • 摘录 API 文档到本地

第七章：最佳实践与技巧 💡

7.1 文档命名规范

为了让 Claudian 更好地理解你的 Vault 结构，建议遵循一致的命名：

分类/文件名.md

示例：
AI/AI名词释义.md
K8s/使用 kubeadm 创建集群.md
K8s/使用 kubeadm 创建集群/未命名.md   ← 子文件夹关联

7.2 YAML Frontmatter 最佳实践

每个笔记都应该包含 Frontmatter，这是 Claudian 自动添加的：

---
title: "笔记标题"
description: "简短描述"
created: 2026-05-31
updated: 2026-05-31
tags:
  - tag1
  - tag2
aliases:
  - 别名1
  - 别名2
---

为什么重要？

  • Obsidian 搜索依赖标签和别名
  • Dataview 插件依赖 Frontmatter 字段
  • 图谱视图的过滤依赖标签

7.3 Wiki-link 使用策略

Claudian 会在笔记之间自动建立链接：

✅ 正确的做法：
  - [[AI名词释义]]          → 链接到另一篇笔记
  - [[AI名词释义#深度学习]]  → 链接到具体标题
  - [[../cloud-native/kubernetes/使用 kubeadm 创建集群/使用 kubeadm 创建集群]] → 跨文件夹链接
  - 在回答中自动使用 [[]] 格式，可点击跳转

❌ 避免的做法：
  - AI名词释义.md（纯文本，不可点击）
  - /绝对路径/AI名词释义.md（路径错误）

7.4 Callout 的使用

Claudian 使用 Obsidian Callout 来区分不同类型的信息：

> [!abstract] 概述
> [!info] 信息
> [!tip] 技巧
> [!warning] 警告
> [!question] 面试题
> [!quote] 引用/原理
> [!check] 清单
> [!important] 核心要点

7.5 DeepSeek 成本控制

模型价格（相对）建议使用场景

deepseek-v4-pro💰💰 中等复杂分析、架构设计、深度写作

省钱技巧：

  • 日常使用 flash 模型
  • 仅对复杂任务切换到 pro
  • 利用 Codex 的会话管理，避免重复上下文加载

第九章：高级技巧 🚀

9.1 插件开发（Codex++）

Codex++ 提供了完整的插件系统，可以创建自定义插件来扩展功能：

.codex-plugin/
├── plugin.json          # 插件元数据（名称、版本、权限等）
├── skills/              # 自定义技能
│   └── SKILL.md         # 技能定义文件
├── scripts/             # 辅助脚本
└── assets/              # 资源文件

plugin.json 示例：

{
  "name": "my-plugin",
  "version": "1.0.0",
  "description": "自定义插件",
  "skills": ["skills/SKILL.md"],
  "permissions": ["filesystem:read", "filesystem:write"]
}

9.2 自定义技能（Skills）

可以通过创建 SKILL.md 文件让 Claudian 具备特定领域的专业知识：

# SKILL.md 示例
## 技能名称：K8s 专家
- 熟练掌握 Kubernetes 架构
- 能诊断集群故障
- 能优化部署配置

当 Codex++ 加载该技能后，Claudian 就会自动具备 K8s 领域的专业行为。

9.3 个人市场（Personal Marketplace）

通过 codex-personal-marketplace.json 可以：

  • 管理个人插件的启用和排序
  • 自定义插件可用性
  • 控制插件加载优先级
{
  "plugins": {
    "codex++": { "enabled": true, "order": 0 },
    "my-plugin": { "enabled": true, "order": 1 }
  }
}

常见问题

问题可能原因解决方案
DeepSeek 模型无响应API Key 错误 / 余额不足检查环境变量 DEEPSEEK_API_KEY 和账户余额
Codex++ 未生效插件未正确加载检查 .codex-plugin/ 目录和 plugin.json
Codex 无法读取 Vault路径配置错误确认 Vault 路径是否正确授予
Claudian 角色没生效SKILL.md 未加载检查技能文件路径是否正确
Obsidian 链接不可点击使用了错误格式使用 [[]] 格式，不要用纯文本路径
文件写入乱码编码问题使用 UTF-8 without BOM 编码
DeepSeek 响应慢网络延迟 / 模型负载高尝试切换到 flash 模型或检查网络
