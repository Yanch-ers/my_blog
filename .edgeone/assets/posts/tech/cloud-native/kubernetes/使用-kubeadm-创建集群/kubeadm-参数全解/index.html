kubeadm init

初始化 Kubernetes 控制平面节点。

kubeadm init [flags]

核心参数

参数默认值说明面试价值
--apiserver-advertise-address默认网关 IPAPI Server 公告的 IP 地址⭐⭐⭐
--apiserver-bind-port6443API Server 绑定的端口⭐⭐
--apiserver-cert-extra-sans[]证书额外的 SAN（域名/IP），多 SAN 用逗号分隔⭐⭐⭐
--cert-dir/etc/kubernetes/pki证书存储目录⭐⭐
--config无kubeadm 配置文件路径（推荐使用）⭐⭐⭐
--control-plane-endpoint无高可用必用，负载均衡器地址 host:port⭐⭐⭐⭐
--cri-socket自动检测CRI 套接字路径，如 unix:///run/containerd/containerd.sock⭐⭐
--dry-runfalse试运行，不实际执行，常用于验证配置⭐⭐
--feature-gates""启用/禁用实验特性，如 --feature-gates="EtcdLearnerMode=true"⭐⭐⭐
--ignore-preflight-errors[]忽略指定的预检错误，如 --ignore-preflight-errors=Swap,NumCPU⭐⭐⭐
--image-repositoryregistry.k8s.io镜像仓库地址，离线部署时修改⭐⭐⭐
--kubernetes-versionstable-1指定 K8s 版本，如 v1.36.0⭐⭐
--node-name主机名节点名称⭐⭐
--pod-network-cidr无Pod 网段 CIDR，必须与 CNI 插件一致⭐⭐⭐⭐
--service-cidr10.96.0.0/12Service 网段 CIDR⭐⭐
--service-dns-domaincluster.localService DNS 域名后缀⭐
--skip-token-printfalse不打印默认 token⭐
--token自动生成指定 bootstrap token⭐⭐
--token-ttl24h0m0stoken 有效时长（0 表示永不过期）⭐⭐
--upload-certsfalseHA 专用，上传控制平面证书供其他节点使用⭐⭐⭐⭐
--v0日志级别，--v=5 查看详细日志用于排错⭐⭐

实战示例

## 📖 最简方式
sudo kubeadm init

## ⚡ 单控制平面（完整参数）
sudo kubeadm init \
  --apiserver-advertise-address=192.168.1.100 \
  --apiserver-cert-extra-sans="k8s.example.com,10.0.0.1" \
  --pod-network-cidr=10.244.0.0/16 \
  --service-cidr=10.96.0.0/12 \
  --kubernetes-version=v1.36.0 \
  --node-name=master-1 \
  --v=5

## 🔧 高可用控制平面首节点
sudo kubeadm init \
  --control-plane-endpoint=192.168.1.20:6443 \
  --apiserver-advertise-address=192.168.1.101 \
  --pod-network-cidr=10.244.0.0/16 \
  --upload-certs

## ⚙️ 离线部署（指定私有镜像仓库）
sudo kubeadm init \
  --image-repository=registry.internal.company.com/k8s \
  --kubernetes-version=v1.36.0 \
  --pod-network-cidr=10.244.0.0/16

## 5. 忽略预检错误（踩坑时用）
sudo kubeadm init \
  --ignore-preflight-errors=Swap,NumCPU,FileAvailable--etc-kubernetes-manifests-etcd.yaml \
  --pod-network-cidr=10.244.0.0/16

## 6. 试运行验证配置
sudo kubeadm init --config=kubeadm-config.yaml --dry-run

参数组合速记

面试口诀

单控制平面记 3 个：--apiserver-advertise-address + --pod-network-cidr + --kubernetes-version

高可用多加 2 个：--control-plane-endpoint + --upload-certs

排错用 3 个：--v=5 + --ignore-preflight-errors + --dry-run

kubeadm join

将节点加入已有集群。

kubeadm join [endpoint:port] [flags]

核心参数

参数默认值说明面试价值
--config无配置文件路径⭐⭐
--control-planefalseHA 专用，将节点作为控制平面加入⭐⭐⭐⭐
--certificate-key无HA 专用，解密控制平面证书的密钥⭐⭐⭐⭐
--cri-socket自动检测CRI 套接字路径⭐⭐
--discovery-file无用于发现集群的配置文件（替代 token）⭐⭐
--discovery-token无发现令牌⭐⭐⭐
--discovery-token-ca-cert-hash无CA 证书的 SHA256 哈希值，验证集群身份⭐⭐⭐⭐
--discovery-token-unsafe-skip-ca-verificationfalse跳过 CA 验证（仅测试用，生产禁止）⭐⭐⭐
--ignore-preflight-errors[]忽略预检错误⭐⭐
--node-name主机名节点名称⭐
--tls-bootstrap-token无TLS 引导令牌⭐⭐
--token无bootstrap token⭐⭐⭐
--v0日志级别⭐

实战示例

## 📖 Worker 节点加入（最常用）
sudo kubeadm join 192.168.1.100:6443 \
  --token <token> \
  --discovery-token-ca-cert-hash sha256:<hash>

## ⚡ 控制平面节点加入（HA）
sudo kubeadm join 192.168.1.20:6443 \
  --token <token> \
  --discovery-token-ca-cert-hash sha256:<hash> \
  --control-plane \
  --certificate-key <key>

## 🔧 指定节点名加入
sudo kubeadm join 192.168.1.100:6443 \
  --token <token> \
  --discovery-token-ca-cert-hash sha256:<hash> \
  --node-name=worker-production-01

## ⚙️ 跳过证书验证（仅开发环境！）
sudo kubeadm join 192.168.1.100:6443 \
  --token <token> \
  --discovery-token-unsafe-skip-ca-verification

## 5. 使用配置文件加入
sudo kubeadm join --config=join-config.yaml

## 6. 忽略预检错误
sudo kubeadm join 192.168.1.100:6443 \
  --token <token> \
  --discovery-token-ca-cert-hash sha256:<hash> \
  --ignore-preflight-errors=Swap

安全警告

--discovery-token-unsafe-skip-ca-verification 严禁在生产环境使用！ 跳过 CA 验证意味着无法确认连接的是否是真正的集群，可能被中间人攻击。

kubeadm token

管理 bootstrap token。

kubeadm token [subcommand] [flags]

子命令

子命令说明常用度
create创建新 token⭐⭐⭐⭐⭐
list列出所有 token⭐⭐⭐⭐
delete删除 token⭐⭐⭐
generate生成但不保存 token⭐⭐

create 参数

kubeadm token create [flags]
参数默认值说明
--config无配置文件路径
--description""token 描述，如 --description="用于 worker 节点加入"
--groups[]token 关联的额外组
--print-join-commandfalse最实用！ 直接打印完整的 join 命令
--ttl24h0m0s有效时长，0 表示永不过期
--usages[signing,authentication]token 用途

实战示例

## 📖 创建 token 并直接打印 join 命令（最常用）
kubeadm token create --print-join-command

## ⚡ 创建永不过期的 token（生产不推荐）
kubeadm token create --ttl=0

## 🔧 创建 2 小时有效期的 token（适合临时使用）
kubeadm token create --ttl=2h --description="临时扩容"

## ⚙️ 列出所有 token
kubeadm token list

## 5. 删除指定 token
kubeadm token delete <token-id>

## 6. 只生成 token 但不保存到集群
kubeadm token generate

实用技巧

最常用的就是 kubeadm token create --print-join-command，一步搞定！ 这个命令会自动生成 token 并拼好完整的 kubeadm join 命令，直接复制粘贴即可。

kubeadm certs

管理集群证书。

kubeadm certs [subcommand] [flags]

子命令

子命令说明面试价值
check-expiration查看所有证书有效期⭐⭐⭐⭐⭐
renew续期证书⭐⭐⭐⭐
generate-csr生成 CSR（证书签名请求）⭐⭐
certificate-key生成用于 HA 的 certificate-key⭐⭐⭐⭐

renew 子参数

kubeadm certs renew [subcommand] [flags]
子命令续期的证书
all续期所有证书
admin.confadmin 客户端证书
apiserverAPI Server 服务端证书
apiserver-etcd-clientAPI Server 连接 etcd 的客户端证书
apiserver-kubelet-clientAPI Server 连接 kubelet 的客户端证书
controller-manager.confController Manager 证书
etcd-healthcheck-clientetcd 健康检查证书
etcd-peeretcd 节点间通信证书
etcd-serveretcd 服务端证书
front-proxy-clientFront Proxy 客户端证书
scheduler.confScheduler 证书

实战示例

## 📖 查看所有证书到期时间（日常巡检）
kubeadm certs check-expiration

## 输出示例：
## [CERTIFICATE]                EXPIRES                  RESIDUAL TIME   CERTIFICATE AUTHORITY
## admin.conf                   May 30, 2027 00:00 UTC   364d            ca
## apiserver                    May 30, 2027 00:00 UTC   364d            ca
## ...

## ⚡ 续期所有证书
kubeadm certs renew all

## 🔧 只续期 API Server 证书
kubeadm certs renew apiserver

## ⚙️ 续期后需要重启控制平面组件
## 修改 Static Pod Manifest 的 annotation 触发滚动更新
## 或直接重启 kubelet
sudo systemctl restart kubelet

## 5. 生成 HA 用的 certificate-key
kubeadm certs certificate-key

## ⚙️  kubectl（续期后需要）
## 如果 admin.conf 更新了，需要重新复制
sudo cp /etc/kubernetes/admin.conf $HOME/.kube/config

重要

kubeadm certs renew all 只续期证书，不会重启组件！

续期后需要：

  1. 重启 kubelet：sudo systemctl restart kubelet
  2. 或者修改 Static Pod Manifest（加个 annotation 触发重建）
  3. 更新用户的 kubeconfig：重新 cp admin.conf ~/.kube/config

kubeadm reset

重置节点，清除所有 kubeadm 安装的数据。

kubeadm reset [flags]
参数默认值说明
--cert-dir/etc/kubernetes/pki证书目录，指定后清除该目录
--config无配置文件路径
--cri-socket自动检测CRI 套接字路径
--forcefalse强制重置，不提示确认
--ignore-preflight-errors[]忽略预检错误

实战示例

## 📖 标准重置
sudo kubeadm reset

## ⚡ 强制重置（不提示确认）
sudo kubeadm reset --force

## 🔧 重置后清理 kubeconfig
rm -rf $HOME/.kube

## ⚙️ 完整重置脚本
sudo kubeadm reset --force
sudo rm -rf /etc/kubernetes/
sudo rm -rf /var/lib/etcd/
sudo rm -rf $HOME/.kube/
sudo iptables -F && sudo iptables -t nat -F && sudo iptables -t mangle -F
sudo systemctl restart containerd

注意

kubeadm reset 不会自动清理 kubelet 配置、容器运行时数据、iptables 规则等。

尤其在控制平面节点上，etcd 数据目录 /var/lib/etcd/ 需要手动删除，否则重新 init 时会报错。

HA 集群升级流程

## ----- 重要：HA 集群必须逐个节点升级！-----

## Step 1：在每个节点上依次执行

## 先 drain 节点（迁移 Pod）
kubectl drain master-1 --ignore-daemonsets --delete-emptydir-data

## 升级 kubeadm
sudo apt-mark unhold kubeadm
sudo apt-get update && sudo apt-get install -y kubeadm=1.36.0-1.1
sudo apt-mark hold kubeadm

## 查看升级计划
sudo kubeadm upgrade plan

## 执行升级
sudo kubeadm upgrade apply v1.36.0

## 升级 kubelet 和 kubectl
sudo apt-mark unhold kubelet kubectl
sudo apt-get update && sudo apt-get install -y kubelet=1.36.0-1.1 kubectl=1.36.0-1.1
sudo apt-mark hold kubelet kubectl
sudo systemctl daemon-reload && sudo systemctl restart kubelet

## 恢复节点调度
kubectl uncordon master-1

## 然后到下一个控制平面节点重复以上步骤

kubeadm alpha

提供 Alpha 级别的功能，可能在未来版本中变更或移除。

kubeadm alpha [subcommand] [flags]
子命令说明
certs证书相关操作（已移入 kubeadm certs，旧版用）
kubeconfig生成 kubeconfig 文件
selfhosting将静态 Pod 转换为自托管 DaemonSet（已废弃）

实用示例

## 生成一个指定用户名的 kubeconfig
kubeadm alpha kubeconfig user --client-name=deploy-bot

注意

kubeadm 1.36 中大部分 alpha 功能已迁移到正式子命令（如 certs）。 不再建议使用 alpha 子命令，直接使用对应的正式子命令即可。

kubeadm 配置文件 YAML

面试重点：为什么要用配置文件而不是命令行参数？

  1. 可重复 — 配置文件可以版本化管理（Git）
  2. 可审计 — 清晰记录集群的初始化配置
  3. 完整 — 命令行参数有限，配置文件的选项更全面
  4. 可迁移 — 配置文件可用于 init、join、upgrade

kubeadm init 配置示例

## kubeadm-config.yaml
apiVersion: kubeadm.k8s.io/v1beta4
kind: InitConfiguration
bootstrapTokens:
  - token: "abcdef.0123456789abcdef"
    description: "kubeadm bootstrap token"
    ttl: "24h"
nodeRegistration:
  name: "master-1"
  criSocket: "unix:///run/containerd/containerd.sock"
  kubeletExtraArgs:
    node-ip: "192.168.1.100"
localAPIEndpoint:
  advertiseAddress: "192.168.1.100"
  bindPort: 6443
---
apiVersion: kubeadm.k8s.io/v1beta4
kind: ClusterConfiguration
kubernetesVersion: "v1.36.0"
controlPlaneEndpoint: "192.168.1.20:6443"   # HA 专用
networking:
  podSubnet: "10.244.0.0/16"
  serviceSubnet: "10.96.0.0/12"
  dnsDomain: "cluster.local"
apiServer:
  certSANs:
    - "k8s.example.com"
    - "10.0.0.1"
  extraArgs:
    authorization-mode: "Node,RBAC"
controllerManager:
  extraArgs:
    bind-address: "0.0.0.0"
scheduler:
  extraArgs:
    bind-address: "0.0.0.0"
imageRepository: "registry.k8s.io"
---
apiVersion: kubelet.config.k8s.io/v1beta1
kind: KubeletConfiguration
cgroupDriver: "systemd"
maxPods: 110
## 使用配置文件初始化
sudo kubeadm init --config=kubeadm-config.yaml --upload-certs

kubeadm join 配置示例

## kubeadm-join-config.yaml
apiVersion: kubeadm.k8s.io/v1beta4
kind: JoinConfiguration
nodeRegistration:
  name: "worker-1"
  criSocket: "unix:///run/containerd/containerd.sock"
  kubeletExtraArgs:
    node-ip: "192.168.1.201"
discovery:
  bootstrapToken:
    token: "abcdef.0123456789abcdef"
    apiServerEndpoint: "192.168.1.20:6443"
    caCertHashes:
      - "sha256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
  tlsBootstrapToken: "abcdef.0123456789abcdef"
---
## 如果是控制平面节点加入，追加以下部分
apiVersion: kubeadm.k8s.io/v1beta4
kind: InitConfiguration
localAPIEndpoint:
  advertiseAddress: "192.168.1.102"
  bindPort: 6443
nodeRegistration:
  name: "master-2"
## 使用配置文件加入
sudo kubeadm join --config=kubeadm-join-config.yaml

配置文件对比：命令行 vs YAML

命令行参数YAML 配置路径备注
--apiserver-advertise-addressInitConfiguration.localAPIEndpoint.advertiseAddress节点级
--pod-network-cidrClusterConfiguration.networking.podSubnet集群级
--control-plane-endpointClusterConfiguration.controlPlaneEndpointHA 专用
--kubernetes-versionClusterConfiguration.kubernetesVersion集群级
--node-nameInitConfiguration.nodeRegistration.name节点级
--cri-socketInitConfiguration.nodeRegistration.criSocket节点级
--cert-dirInitConfiguration.certificateDir节点级
--image-repositoryClusterConfiguration.imageRepository集群级
--service-cidrClusterConfiguration.networking.serviceSubnet集群级
--tokenInitConfiguration.bootstrapTokens[0].token节点级
